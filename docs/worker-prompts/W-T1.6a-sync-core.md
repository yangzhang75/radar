## 实现要求（纯函数/纯逻辑，注入模拟时间，不用真实时钟/线程）
- **辐条排序/缓冲**：按 `sequenceNumber`(0~4095循环)与方位组织一圈辐条；检测丢失/乱序/重复(序号回绕处理)；提供"丢包率/缺口"统计供降级判断。
- **时间同步/对齐**：把不同更新率的数据(本船高频、目标、状态)按时间戳对齐到统一时刻，供 UI 一致渲染；高延迟下的时间偏移补偿策略(纯计算)。
- **断线检测/重连策略**：纯策略——给定"最后收包时间/当前tick/各通道活跃度"，判定通道是否中断(→触发 3002 通信中断报警标志)、何时重连、退避(backoff)。暴露 `onTick(now)`/`onPacket(channel,ts)` 纯接口，由 Service 驱动。
- **缓冲/背压**：低带宽下的丢弃策略(如旧辐条可丢、状态/报警不可丢)，有界缓冲。
- **去重**：跨重传的重复帧识别。

## 验证
